# 角色设计助手

按照《我在火影玩数值暴力》项目规范，设计新角色或完善现有角色设定。

## 权威优先规则（必须遵守）

本 Skill 中的规则摘要仅供参考，实际执行时必须：

1. **先读取原文件** — 角色传记/创作手册/时间线必须完整读取
2. **原文件优先** — 当本 Skill 内容与原文件冲突时，**以原文件为准**
3. **版本校验** — 执行前检查原文件版本号，若版本更新则必须重新读取
4. **禁止硬编码覆盖** — 本 Skill 的等级档位摘要不能替代创作手册的完整规则

## 三层优先级（不可违反）

角色设计必须遵守以下优先顺序：

1. **已落地正文** — 设计不能推翻已写内容
2. **生产大纲** — 角色作用必须符合大纲定位
3. **世界圣经** — 等级能力不超过档位规则

## 角色设计流程

### 第一步：确认角色定位

向用户询问以下核心问题：

1. **角色类型**：原著角色扩展 / 新增OC角色 / 蝴蝶效应衍生角色
2. **故事作用**：情感锚点 / 主题镜子 / 压力源 / 剧情工具人 / 观众型震惊角色
3. **出场范围**：单篇配角 / 线级配角 / 主线角色
4. **等级上限**：参照创作手册等级档位规则

### 第三步：校验设计

设计完成后，逐项检查：

- [ ] 编号是否与目录索引一致
- [ ] 等级能力是否在档位规则范围内

- [ ] 是否有明确的OOC风险点提醒

## 角色类型专项指南

### A. 原著角色扩展设计

**适用情况**：原著角色在本作中的重新定位或细节补充

**设计要点**：
- 必须尊重原著核心性格
- 蝴蝶效应导致的改变必须有合理因果链
- 记录"与本作的关联"部分，说明与原著的差异点

**示例检查**：
- [ ] 原著核心性格是否保留
- [ ] 改变是否有蝴蝶效应支撑
- [ ] 是否标注了"创作注意事项"防止OOC

### B. 新增OC角色设计

**适用情况**：完全原创的新角色

**设计要点**：
- 必须有明确的故事作用（不能是"为了好看而加"）
- 等级不能无理由超过同档位原著角色
- 必须设计"致命缺陷"防止角色过于完美

**等级档位参考**（来自创作手册）：

| 等级段 | 能做到什么 | 做不到什么 |
|--------|------------|------------|
| Lv.1-10 | 感知、基础破局、近身小胜 | 不能硬改命 |
| Lv.11-20 | D/C级任务稳定 | 不能正面扛上忍 |
| Lv.21-35 | 中忍对抗 | 不能长期越级碾压 |
| Lv.36-50 | 上忍战边缘 | 不能改写因果链 |
| Lv.51-65 | 可干预笼中鸟级逻辑锁 | 不能无代价连救多人 |
| Lv.66-85 | 晓级战主力 | 不能无视组织战 |
| Lv.86-100 | 影级/尾兽战 | 不能抹掉世界规则 |

### C. 蝴蝶效应衍生角色设计

**适用情况**：因主角行为导致的原著角色命运改变

**设计要点**：
- 必须记录"蝴蝶效应起点"：哪个事件导致变化
- 必须标注"原著命运"与"本作命运"的差异
- 必须评估"改命代价"：这个改变消耗了什么

**输出增加部分**：

```markdown
## 蝴蝶效应分析

### 原著命运
{原著中该角色的结局}

### 本作命运
{本作中该角色的变化}

### 蝴蝶效应起点
{哪个事件导致了命运改变}

### 改命代价
{这个改变消耗了什么代价（掉级/念气紊乱/后续风险）}
```

## 常见角色定位类型

| 类型 | 功能 | 设计要点 |
|------|------|----------|
| **情感锚点** | 读者情感寄托 | 性格必须有可爱/可敬之处，悲剧时有痛感 |
| **主题镜子** | 与主角理念对撞 | 必须有与主角相反的价值观，但非纯粹反派 |
| **压力源** | 给主角制造困难 | 必须有足够实力或地位，压迫感真实 |
| **观众型角色** | 提供震惊反应 | 性格要鲜明，反应要有辨识度 |
| **剧情工具人** | 推动事件 | 必须有明确功能，但性格不能过于工具化 |

## 执行流程

1. **询问用户**：确认角色类型、定位、出场范围
2. **读取参考文件**：目录索引、创作手册、时间线
3. **设计核心要素**：按模板输出完整设定
4. **校验设计**：逐项检查清单
5. **输出文件**：按编号命名保存到 `资料库/人物传记/`
