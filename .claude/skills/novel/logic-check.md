# 小说逻辑检测助手

对《我在火影玩数值暴力》的已写章节进行全方位逻辑漏洞检测，输出问题清单与修复建议。

## 权威优先规则（必须遵守）

本 Skill 中的规则摘要仅供参考，实际检测时必须：

1. **先读取原文件** — 创作手册/时间线/角色传记必须完整读取
2. **原文件优先** — 当本 Skill 的检测规则与原文件冲突时，**以原文件为准**
3. **版本校验** — 检测前检查原文件版本号，若版本更新则必须重新读取
4. **禁止硬编码覆盖** — 本 Skill 的等级档位表/角色性格速查不能替代原文件的完整规则

**权威文件清单**：
| 文件 | 版本检查点 | 用途 |
|------|------------|------|
| `创作手册_完整版_456章_v2.0.0.md` | 文件名版本号 | 世界规则最终准绳 |
| `创作手册_1-150章生产大纲.md` | 文件首行版本标注 | 章节功能依据 |
| `火影忍者时间线.md` | 文件开头纪元说明 | 时间锚点最终准绳 |
| `资料库/人物传记/*.md` | 文件末尾更新日期 | 人物性格最终准绳 |

**重要警告**：本 Skill 内嵌的"等级档位规则表"和"原著角色性格速查"是**静态摘要**，仅用于快速参考。实际检测时必须对照创作手册和人物传记原文件进行校验，避免因原文件更新导致检测规则过时。

## 检测维度总览

| 维度 | 检测内容 | 严重等级 |
|------|----------|----------|
| **世界观规则** | 等级档位、系统规则、命运节点、改命代价 | 🔴 高 |
| **时间线锚点** | 原著时间、本项目时间锚点、章节时间连续性 | 🔴 高 |
| **剧情逻辑** | 章节衔接、伏笔回收、因果关系、蝴蝶效应链 | 🟡 中 |
| **人物OOC** | 性格一致性、能力边界、行为动机、关系设定 | 🔴 高 |
| **场景合理性** | 空间锚点、战斗链条、身体反馈、技能效果 | 🟡 中 |
| **情绪曲线** | 憋屈指数、爽点密度、钩子力度 | 🟢 低 |

检测前必须读取以下文件：

1. **创作手册**: `创作手册_完整版_456章_v2.0.0.md` — 世界观规则、等级档位、角色线速查

## 检测流程

### 第一步：确认检测范围

向用户询问：
1. 检测对象（单章/多章/全篇）
2. 检测重点（全部维度/特定维度）
3. 检测深度（快速扫描/深度分析）

### 第二步：读取必要文件

按顺序读取创作手册、时间线、角色传记、待检测章节。

### 第三步：逐维度检测

按以下检测清单逐项扫描：

---

## A. 世界观规则检测

### A1. 等级档位校验

**检测项**：
- 主角当前等级是否明确标注
- 本章战斗表现是否超出档位上限
- 能力使用是否符合"能做到/做不到"边界

**等级档位规则表**（来自创作手册）：

| 等级段 | 能做到什么 | 做不到什么 |
|--------|------------|------------|
| Lv.1-10 | 感知、基础破局、近身小胜 | 不能硬改命 |
| Lv.11-20 | D/C级任务稳定 | 不能正面扛上忍 |
| Lv.21-35 | 中忍对抗、短时爆发 | 不能长期越级碾压 |
| Lv.36-50 | 上忍战边缘 | 不能改写大型因果链 |
| Lv.51-65 | 可干预笼中鸟级逻辑锁 | 不能无代价连救多人 |
| Lv.66-85 | 晓级战主力 | 不能无视组织战 |
| Lv.86-100 | 影级/尾兽战 | 不能抹掉世界规则 |

**检测输出格式**：
```markdown
#### A1. 等级档位校验
- **当前等级**：Lv.{X}
- **本章战斗表现**：{简要描述}
- **是否符合档位**：✅符合 / ❌超标
- **超标描述**（如有）：{具体说明超出哪项限制}
- **修复建议**：{如何调整战斗描写或等级标注}
```

### A2. 系统规则校验

**检测项**：
- 系统UI提示是否在高潮战斗/改命节点弹出
- 能力成长是否先有现实反馈再给系统提示
- 改命是否有真实代价且不在当章抹平

**系统规则清单**：
- [ ] 高潮战斗必须有高调系统提示
- [ ] 能力成长不能"睡一觉满血回档"
- [ ] 改命代价必须延续到后续章节

**检测输出格式**：
```markdown
#### A2. 系统规则校验
- **系统提示出现次数**：{X}次
- **提示位置是否合理**：✅合理 / ❌缺失或不当
- **能力成长反馈链**：✅完整 / ❌跳过现实反馈
- **改命代价延续**：✅延续 / ❌当章抹平
- **问题清单**：
  - {问题描述}
- **修复建议**：{具体修改方案}
```

### A3. 命运节点校验

**检测项**：
- 命运节点是否在"原著强因果链 + 主角触碰核心人物"时弹出
- 是否有不该弹节点的情况（普通濒死）
- 节点选项是否符合三种类型

**命运节点规则**：
- 只在改变卷级走向的死亡/转折弹出
- 选项只有：立即干预 / 延后干预 / 不干预
- 节点越大，代价越高

**检测输出格式**：
```markdown
#### A3. 命运节点校验
- **本章是否出现命运节点**：是/否
- **触发条件是否合理**：✅合理 / ❌不当触发
- **不当触发描述**（如有）：{为何不该弹节点}
- **选项是否符合规范**：✅符合 / ❌偏离
- **代价计算是否合理**：✅合理 / ❌过低或无效
- **修复建议**：{具体调整方案}
```

### A4. 改命代价校验

**检测项**：
- 改命是否伴随掉级/念气紊乱/后续风险
- 掉级恢复是否靠战斗/训练/高压突破
- 连续改命是否叠加副作用

**改命代价规则**：
- 默认代价：临时掉级 + 念气紊乱 + 后续风险
- 恢复必须靠后续努力，不能睡一觉回档
- 连续改命叠加：技能锁死、感知失真、回路迟滞

**检测输出格式**：
```markdown
#### A4. 改命代价校验
- **本章是否有改命行为**：是/否
- **代价类型**：{掉级/念气紊乱/技能封印}
- **代价延续性**：✅延续 / ❌当章抹平
- **恢复方式**：{战斗/训练/高压突破} / ❌无代价恢复
- **副作用叠加**：✅有 / ❌无
- **修复建议**：{具体调整方案}
```

## B. 时间线锚点检测

### B1. 原著时间线校验

**检测项**：
- 本章事件在原著时间线上的位置是否合理
- 是否与原著重大事件冲突（如鸣人毕业时间）
- 角色年龄是否符合原著设定

**原著时间锚点**（来自创作手册）：

| 锚点 | 原著时间 | 本书用途 |
|------|----------|----------|
| 鸣人毕业/第七班成立 | 木叶62年3月 | 前期木叶篇基线 |
| 波之国任务 | 木叶62年4月 | 35-50章锁定 |
| 鸣人回归木叶 | 木叶65年3月 | 151章后疾风传 |
| 辉夜复活/封印 | 木叶68年大战第3天 | 终局区 |

**检测输出格式**：
```markdown
#### B1. 原著时间线校验
- **本章时间锚点**：{木叶XX年X月}
- **原著对应时间**：{原著事件}
- **是否冲突**：✅无冲突 / ❌存在冲突
- **冲突描述**（如有）：{具体说明}
- **角色年龄校验**：
  - {角色名}：{当前年龄}，原著设定{原著年龄}，✅/❌
- **修复建议**：{具体调整方案}
```

### B2. 章节时间连续性校验

**检测项**：
- 与上一章的时间是否连续或有明确间隔标注
- 是否存在时间断层（上章晚上→本章早上无说明）
- 场景内的时间推进是否合理

**检测输出格式**：
```markdown
#### B2. 章节时间连续性校验
- **上章时间锚点**：{上章结尾时间}
- **本章开篇时间**：{本章开头时间}
- **时间衔接**：✅连续或有标注 / ❌断层
- **断层描述**（如有）：{缺少什么时间说明}
- **场景内时间推进**：✅合理 / ❌不合理
- **修复建议**：{补充时间标注或调整事件顺序}
```

## C. 剧情逻辑检测

### C1. 章节衔接校验

**检测项**：
- 本章开篇是否承接上章的人物位置和状态
- 是否存在镜头跳跃（上章在A地→本章突然在B地）
- 上章悬念/情感债是否在本章回应或延续

**检测输出格式**：
```markdown
#### C1. 章节衔接校验
- **上章结尾状态**：
  - 地点：{上章地点}
  - 人物：{上章人物位置/动作}
  - 悬念：{上章钩子}
- **本章开篇状态**：
  - 地点：{本章地点}
  - 人物：{本章人物位置}
- **衔接评估**：✅平滑衔接 / ❌镜头跳跃 / ❌未回应悬念
- **跳跃描述**（如有）：{具体说明}
- **修复建议**：{补充过渡描写或调整场景}
```

### C2. 伏笔回收校验

**检测项**：
- 本章是否埋下新伏笔
- 是否有旧伏笔在本章回收
- 未回收伏笔是否有追踪记录

**检测输出格式**：
```markdown
#### C2. 伏笔回收校验
- **本章新埋伏笔**：
  - {伏笔描述}（预计回收章节：{X}章）
- **本章回收伏笔**：
  - {伏笔描述}（来源章节：{X}章）
- **未回收伏笔清单**（跨章追踪）：
  - {伏笔描述}（来源：{X}章，待回收）
- **伏笔遗忘风险**：✅无 / ❌有长期未回收伏笔
- **修复建议**：{回收计划或补充伏笔呼应}
```

### C3. 因果链条校验

**检测项**：
- 事件A导致事件B的因果链是否完整
- 是否存在"无因果跳跃"（A→C跳过B）
- 蝴蝶效应是否有合理传导路径

**检测输出格式**：
```markdown
#### C3. 因果链条校验
- **本章关键因果链**：
  - {事件A} → {事件B} → {事件C}
- **因果缺失**（如有）：{缺少的中间环节}
- **蝴蝶效应传导**：✅合理 / ❌跳跃
- **修复建议**：{补充中间环节或调整因果顺序}
```

## D. 人物OOC检测

### D1. 主角性格校验

**检测项**：
- 战斗时是否从容不迫（无慌张描写）
- 日常是否有毒舌/黑色幽默（无说教）
- 处事是否符合"内圣外王+斩草除根"原则

**主角人设规则**：
- 战斗：从容、降维压制、不吃力
- 日常：毒舌、高逼格吐槽、护短
- 处事：不隐忍、不圣母、摧枯拉朽

**检测输出格式**：
```markdown
#### D1. 主角性格校验
- **战斗表现**：✅从容 / ❌有慌张描写
- **日常表现**：✅毒舌护短 / ❌说教/圣母
- **处事原则**：✅斩草除根 / ❌隐忍/圣母
- **OOC段落定位**（如有）：
  - 行号：{X}，问题：{具体描述}
- **修复建议**：{删除/修改OOC描写}
```

### D2. 原著角色性格校验

**检测项**：
- 原著角色（鸣人、佐助、卡卡西等）是否性格崩塌
- 是否有合理的"蝴蝶效应扭曲"
- 行为动机是否符合原著性格基础

**原著角色性格速查**：
- 鸣人：乐观、不服输、重视羁绊、笨蛋属性
- 佐助：冷酷、复仇驱动、骄傲、内心矛盾
- 卡卡西：慵懒、可靠、重视同伴、有阴暗面
- 小李：热血、执着、单纯、努力至上
- 宁次：宿命论、冷静、骄傲、内心压抑

**检测输出格式**：
```markdown
#### D2. 原著角色性格校验
- **出场原著角色**：{角色列表}
- **性格一致性检查**：
  - {角色名}：✅一致 / ❌OOC
- **OOC描述**（如有）：
  - {角色名}：{具体问题}
- **蝴蝶效应合理性**：✅合理扭曲 / ❌无依据崩塌
- **修复建议**：{调整描写或补充扭曲原因}
```

### D3. 能力边界校验

**检测项**：
- 角色使用的技能是否符合当前设定
- 是否超出角色已掌握的忍术范围
- 战力表现是否与角色等级匹配

**检测输出格式**：
```markdown
#### D3. 能力边界校验
- **本章使用技能**：
  - {角色名}：{技能列表}
- **技能设定检查**：
  - {技能名}：✅已掌握 / ❌超出设定
- **战力匹配度**：✅匹配 / ❌超标或过低
- **修复建议**：{调整技能使用或补充学习过程}
```

## E. 场景合理性检测

### E1. 空间锚点校验

**检测项**：
- 战斗场景是否有地形、距离、遮挡、高差描写
- 场景切换是否有地点变化标注
- 人物位置是否清晰

**检测输出格式**：
```markdown
#### E1. 空间锚点校验
- **本章战斗场景数**：{X}
- **空间锚点检查**：
  - 场景1：✅有地形/距离/遮挡 / ❌缺失
- **场景切换标注**：✅清晰 / ❌模糊
- **人物位置明确性**：✅明确 / ❌混乱
- **修复建议**：{补充空间描写}
```

### E2. 战斗链条校验

**检测项**：
- 是否包含完整链条：站位→起手→变招→命中→身体反馈→场面后果
- 是否有"只报技能名不写动作"问题
- 终结动作是否有半拍蓄压

**战斗镜头链规则**：
```
站位 → 起手 → 变招 → 命中/落空 → 身体反馈 → 场面后果
```

**检测输出格式**：
```markdown
#### E2. 战斗链条校验
- **战斗段落定位**：行号{X}-{Y}
- **链条完整性**：
  - 站位：✅/❌
  - 起手：✅/❌
  - 变招：✅/❌
  - 命中：✅/❌
  - 身体反馈：✅/❌
  - 场面后果：✅/❌
- **缺失环节**：{具体说明}
- **技能名空洞问题**：✅无 / ❌有
- **终结蓄压**：✅有 / ❌无
- **修复建议**：{补充缺失环节描写}
```

### E3. 身体反馈校验

**检测项**：
- 是否有物理反馈描写（手麻、视野晃、呼吸乱等）
- 反馈是否与战斗强度匹配
- 是否只用"他很强"等空洞描述

**身体反馈类型**：
- 手麻、手臂酸痛、握力下降
- 视野晃动、视线模糊、眼前发黑
- 呼吸紊乱、气喘、胸闷
- 脚下打滑、腿软、站不稳
- 肋骨发闷、内脏震动、骨骼疼痛

**检测输出格式**：
```markdown
#### E3. 身体反馈校验
- **身体反馈出现次数**：{X}次
- **反馈类型**：{列举出现的反馈类型}
- **与战斗强度匹配**：✅匹配 / ❌过轻或过重
- **空洞描述问题**：✅无 / ❌有"很强"类空洞词
- **修复建议**：{补充具体身体反馈}
```

## F. 情绪曲线检测

### F1. 憋屈指数校验

**检测项**：
- 是否有连续超过800字的压抑描写
- 憋屈段落是否过于集中
- 是否有爽点对冲

**憋屈段落定义**：
- 主角被压制、被误解、被羞辱
- 连续失败、无力感、绝望感
- 正面角色死亡或受伤

**检测输出格式**：
```markdown
#### F1. 憋屈指数校验
```

### F2. 爽点密度校验

**检测项**：
- 本章爽点出现次数
- 爽点类型分布是否合理
- 爽点密度是否符合"每10章爆点配比"

**爽点类型**：
| 类型 | 说明 |
|------|------|
| 机制爽点 | 看穿、反制、数值压制、打脸 |
| 情感爽点 | 兄弟线、师徒线、被认可、被追随 |
| 世界爽点 | 高层关注、制度裂缝、血脉门槛揭开 |
| 命运爽点 | 看到能不能救、为什么救不了 |

**爽点配比要求**（每10章）：
- 机制爽点：3次
- 情感爽点：2次
- 世界爽点：1次
- 命运爽点：1次

**检测输出格式**：
```markdown

- **爽点分布**：✅均匀 / ❌集中或缺失
- **修复建议**：{补充爽点类型建议}
```

## 执行流程

1. **确认检测范围** — 单章/多章/特定维度
2. **读取必要文件** — 创作手册、时间线、角色传记、章节正文
3. **逐维度检测** — 按检测清单逐项扫描
4. **输出检测报告** — 按综合报告格式输出
5. **提供修复建议** — 针对每个问题给出具体修改方案
