## 设计类型

根据用户需求，选择以下设计模式：

### A. 章节细纲设计

**输出格式**（严格遵循 `claude-output-convention.md` 章节细纲模板）：

```markdown
### 第[X]章：[标题]（口径：结果 > 冲突 > 情绪 > 概念）

- **本章功能**：（在卷/篇章中的结构性任务）
- **核心冲突**：（一句话核心矛盾）
- **上章衔接**：（必须平滑承接，绝不允许镜头跳跃）
- **章节爽点/钩子**：（最能刺激读者情绪的点）
- **必须承接到下一章的情感或信息**：（情感债或信息锚点）
- **场景推进表**：
  - 场景1：[环境/起手动作/氛围营造]
  - 场景2：[冲突展开/金手指互动/配角反应]
  - 场景3：[高潮爆发/系统高调UI提示/机制碾压]
  - 场景4：[战后反馈/环境余波/围观者反应]
  - 场景5：[主角的决策或内心转变]
  - 场景6（章尾）：[钩子挂起/下章悬念/情感留白]
- **镜头/叙事手法提示**：（实际落稿指导）
```

**设计检查清单**：
- [ ] 标题符合"结果 > 冲突 > 情绪 > 概念"口径
- [ ] 核心冲突在"理念冲突速查卡"中有对应或合理扩展
- [ ] 爽点密度符合"每10章爆点配比"要求
- [ ] 系统UI提示设计合理（如 `【破招判定成功】`）
- [ ] 上章衔接具体明确，不是模糊表述

### C. 冲突场景设计

**设计要素**：

1. **对立命题** — 双方的理念差异
2. **首次碰撞点** — 第一次正面交锋的场景
3. **输赢设计** — 第一次交锋的结果
4. **情感代价** — 冲突给双方带来的心理影响
5. **系统介入** — DNF机制如何碾压忍术体系

**输出格式**：

```markdown
### 冲突场景：[参与者A] vs [参与者B]

- **对立命题**：[A的理念] vs [B的理念]
- **冲突场景**：
  - 地点：
  - 触发条件：
  - 双方初始状态：
- **交锋过程设计**：
  - 起手：
  - 变招：
  - 关键破局点：
  - 系统UI提示：（如 `【霸体护甲生成】`）
- **结果设计**：
  - 表面输赢：
  - 实际代价：
- **情感回响**：（对后续剧情的影响）
```

### D. 命运节点设计

**设计原则**：

- 只在"原著强因果链 + 主角触碰核心人物"时弹出
- 必须包含系统高调提示（如 `【警告！S级命运节点挂起】`）
- 节点越大，代价越高

**输出格式**：

```markdown
### 命运节点：[节点名称]

- **触发条件**：（原著强因果链 + 主角介入点）
- **系统提示设计**：
  - 警告级别：（S/A/B级）
  - UI提示文案：（如 `【启动最后生机锁定干预】`）
- **选项设计**：
  - 立即干预 → 代价：
  - 延后干预 → 代价：
  - 不干预 → 结果：
- **代价计算**：
  - 掉级幅度：
  - 念气紊乱程度：
  - 后续战斗风险提升：
- **成功后的情感账单**：（人活下来，账还在）
```

## 写作技法提示

从生产大纲的"写法工具卡"提取关键技法：

### 战斗镜头规则
- `站位 → 起手 → 变招 → 命中/落空 → 身体反馈 → 场面后果`
- 必须有空间锚点：地形、距离、遮挡物、高差
- 必须有身体反馈：手麻、视野晃、呼吸乱、脚下打滑
- 终结动作前必须有半拍蓄压

### 情绪回响规则
- 悲情靠动作停顿、对话断裂、环境冷感
- 不靠大段自怜
- 情绪章必须落到"这一次决定因此变了什么"

### 无敌流高逼格规则
- 主角战斗从容不迫，带有降维压制感
- 配角要当人写，不做挨打木桩
- 前期日常羁绊越有烟火气，后期修罗手段爆发越有正当性

## 执行流程

1. **询问用户**：确认设计类型（章节细纲/角色设定/冲突场景/命运节点）
2. **读取参考文件**：根据设计类型读取必要的创作手册章节
3. **校验优先级**：确认不违反三层文档体系
4. **输出设计**：按对应模板格式输出
5. **自查清单**：逐项检查设计是否符合规范
