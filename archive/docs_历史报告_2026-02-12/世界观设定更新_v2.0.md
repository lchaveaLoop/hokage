## 🔧 一、系统能级限制设定（补充）

### 1.1 当前问题

**矛盾点**: 第1章"系统能级暂时锁定为Lv.1" vs 第50章"升至Lv.50"未说明限制解除条件

### 1.2 新增设定

#### **系统能级解锁机制**

```
【系统设定：能级解锁规则】

初始状态（第1章）:
- 系统能级: 暂时锁定为 Lv.1
- 解锁方式: 随载体适应度逐步解锁

解锁节点:
- Lv.1 → Lv.20（第7章转职）:
  解锁条件: 载体完成转职任务（散打武魂试炼）
  解锁奖励: 开启【技能树】、【觉醒系统】

- Lv.20 → Lv.50（第100章一觉）:
  解锁条件: 载体击败【影级】敌人或完成【拯救重要角色】任务
  解锁奖励: 开启【第一次觉醒】、【规则级技能】

- Lv.50 → Lv.75（第200章二觉）:
  解锁条件: 载体净化【1000名伪装者】或击败【使徒分身】
  解锁奖励: 开启【第二次觉醒】、【极武圣形态】

- Lv.75 → Lv.100（第300章三觉）:
  解锁条件: 载体击败【使徒本体】或阻止【世界级危机】
  解锁奖励: 开启【第三次觉醒】、【归元境界】

- Lv.100+（第400章+）:
  解锁条件: 突破系统限制,超越规则
  解锁奖励: 【归元境界】、【因果判定】、【时空破】
```

## 🩺 二、念气治疗设定（明确代价）

### 2.1 当前问题

**矛盾点**: 第1章"无法主动释放治疗类技能" vs 第53章"念气治疗保住三代生命"

### 2.2 新增设定

#### **念气治疗机制**

```
【系统设定：念气治疗规则】

基础限制（第1-50章）:
- 主角无法主动释放治疗类技能
- 原因: 散打职业定位为【纯输出】,无治疗技能

临时解锁条件:
- 当【重要角色生命值<10%】时
- 主角【等级≥Lv.30】
- 主角【主动选择消耗代价】

治疗代价:
- 方式A: 消耗主角10%当前经验值
- 方式B: 消耗主角50%当前查克拉
- 方式C: 消耗系统商城道具【神圣生命水】

治疗效果:
- 目标生命值恢复至30%
- 目标状态稳定,不再继续恶化

副作用:
- 主角等级下降1级（若选择方式A）
- 主角查克拉枯竭24小时（若选择方式B）
- 系统商城道具一次性消耗（若选择方式C）
```

#### **应用场景**

**第53章（救三代）修改版**:
```

系统UI提示:
【检测到重要角色生命垂危】
【是否消耗10%经验值进行念气治疗？】
【警告: 等级将从Lv.35降至Lv.34】

李武: "治疗。"
系统: 【确认】
【消耗10%经验值】
【念气治疗发动】

金色的念气环绕三代
三代生命值稳定在30%
系统UI: 【治疗成功】【等级降至Lv.34】

李武内心: "1级换一条命...值得。"
```

## 👁️ 三、血条可见性设定（明确范围）

### 3.1 当前问题

**矛盾点**: 第2章"仅主角可见(UI层)" vs 第144章"小李看到哥哥头顶的金色光晕"

### 3.2 新增设定

#### **血条可见性规则**

```
【系统设定：血条可见性】

UI层（仅主角可见）:
- 元素: 能级颜色（白/蓝/紫/粉/橙/金）
- 元素: 查克拉状态颜色渐变（蓝→紫→红）
- 元素: 弱点红点（仅李武独见）
- 设定依据: 【至高至善】位面舞王系统对现实规格的数字化投射
- 表现: 在他人看来,这是李武超乎常人的洞察力

物理层（所有人可见）:
- 元素: 金色的念气波
- 元素: 招式激发的冲击波（气浪/激波）
- 元素: 霸体状态下的金属感色泽
- 元素: 瞬步留下的残影
- 设定依据: 能量（念气/查克拉）在现实世界的物理干涉
- 表现: 必须符合物理特性（念气波有推力,冲击波会震碎地面）

禁止出现（任何人不可见）:
- 具体数字（如"生命值:20%"）
- 百分比条
- 游戏术语（如"死线""斩杀"）
- 血条层数数字
```

#### **修改示例**

**第144章修改版**:

**原版**:
```
小李: "哥哥...你的头顶,有光。"
（错误: 小李不该看到血条）
```

**新版**:
```
小李感受到温暖的念气
小李: "哥哥...你的力量,好温暖。"
（正确: 小李感受到念气,而非看到血条）

或者:
小李看到哥哥周围金色的念气波纹
小李: "哥哥...你全身都在发光！"
（正确: 小李看到物理层的念气波纹）
```

## ⚔️ 四、判定优先级设定（补充等级）

### 4.1 当前问题

**矛盾点**: 创作手册"无敌>抓取>霸体>普通" vs 第70章"粉碎判定"级别未明确

### 4.2 新增设定

#### **判定等级对照表**

```
【系统设定：判定优先级】

第0级: 【概念级】（最高）
- 技能: 尸鬼封尽、地爆天星、无限月读
- 特点: 规则级存在,无视常规判定
- 示例: 三代尸鬼封尽无法被霸体抵挡

第1级: 【神级】
- 技能: 武神强踢、须佐能乎完全体
- 特点: 强制清空视野内所有低阶单位
- 示例: 第165章武神强踢踢歪神罗天征

第2级: 【规则级】
- 技能: 极武霸皇拳、求道玉
- 特点: 规则层面压制,部分可对抗
- 示例: 极武霸皇拳碰撞须佐能乎

第3级: 【无敌判定】
- 技能: 念气罩、蹲伏、觉醒演出时间
- 特点: 在持续时间内完全免疫伤害
- 示例: 念气罩挡下尾兽玉

第4级: 【绝对抓取】（=粉碎判定）
- 技能: 背摔、折颈、寸拳（粉碎模式）
- 特点: 无视霸体,强制控制
- 示例: 第70章寸拳粉碎君麻吕骨片

第5级: 【强霸体】
- 技能: 武神步冲刺、霸体护甲、八门遁甲
- 特点: 硬扛伤害不倒地,但可被抓取技破除
- 示例: 霸体硬钢千鸟

第6级: 【普通判定】
- 技能: 普通忍术、手里剑、体术平A
- 特点: 常规攻击,无特殊优先级
- 示例: 普通查克拉手术刀
```

## 💀 五、改命代价设定（新增）

### 5.1 当前问题

**缺失点**: 改变原著命运缺乏代价,降低选择重量感

### 5.2 新增设定

#### **改命代价系统**

```
【系统设定：改命代价】

定义:
- 改变"原著死亡角色"的命运,需要支付代价

代价规则:
- 重要角色（三代、鼬、宁次、自来也等）: 消耗主角1级或10%经验值
- 次要角色（君麻吕、再不斩、白等）: 消耗主角5%经验值
- 普通角色: 无消耗

改命方式:
- 方式A: 主动改命（主角发起）
  - 需要先支付代价
  - 成功率: 100%
- 方式B: 被动改命（战斗中救下）
  - 事后补足代价
  - 成功率: 依情况而定
- 方式C: 提供选择（角色自己决定）
  - 主角提供机会,角色自己选择
  - 无需支付代价

应用示例:
- 第53章救三代: 主动改命,消耗1级（Lv.35→Lv.34）
- 第117章救千代: 战斗中救下,事后消耗5%经验
- 第136章救鼬: 提供选择,鼬选择活下去,无代价
```

## 📚 七、设定管理机制

### 7.1 设定冲突检测标准

```
【设定管理规则】

检测维度:
1. 时间线一致性
   - 检查点: 所有涉及时间流逝的描述
   - 标准: 必须与时间线对照表一致

2. 能力逻辑一致性
   - 检查点: 所有涉及主角能力的描述
   - 标准: 必须与技能数据库一致

3. 角色动机合理性
   - 检查点: 所有重要角色的关键行为
   - 标准: 必须有明确的动机铺垫

4. 世界观设定自洽性
   - 检查点: 所有涉及世界观的内容
   - 标准: 必须与创作手册一致

检测频率:
- 写作时: 作者自检
- 每章完成后: 编辑初审
- 每10章: 设定全面审查
- 每50章: 设定大修订
```

### 7.2 设定变更审批流程

```
【设定变更流程】

提议:
- 作者发现设定问题或需要新增设定
- 填写《设定变更申请表》

评估:
- 编辑评估变更必要性
- 评估对现有内容的影响
- 给出修改建议

审批:
- 重要设定变更: 需要作者+编辑双重确认
- 一般设定变更: 编辑审批即可
- 紧急设定修复: 直接修复,事后补申请

实施:
- 更新创作手册
- 更新相关章节
- 记录变更历史

验证:
- 交叉验证（作者自审+编辑初审+测试读者）
- 确认无矛盾后,正式生效
```

### 7.3 设定变更记录表

```
【设定变更历史】

格式:
- 变更ID: SVN-YYYYMMDD-序号
- 变更类型: [新增/修改/删除/修正]
- 变更日期: YYYY-MM-DD
- 变更内容: 具体描述
- 影响范围: 受影响的章节/设定
- 变更原因: 审读报告/作者决策/其他
- 审批人: 编辑/作者
- 状态: [提议/审批中/已实施/已验证]

示例:
- 变更ID: SVN-20260206-001
- 变更类型: 修正
- 变更日期: 2026-02-06
- 变更内容: 第61章"三年后"改为"两年半后"
- 影响范围: 第61章,时间线设定
- 变更原因: 审读报告P0级别问题
- 审批人: 编辑
- 状态: 已实施
```
