## 五、使用说明

- **调用方式**：在 Claude Code 对话中直接 `@claude-output-convention.md` 并说明需求，例如：
  > “@claude-output-convention.md 请按‘逻辑漏洞检测模板’分析我第 47 章的内容。”
- **输出后操作**：你拿到的将是纯 Markdown 文本，可直接在 Obsidian、Typora、Word 等软件中编辑，修改后可再次喂给 Claude Code 进行迭代。
- **扩展性**：如需新增模板（如评论区风向预测、金手指系统设计），直接在本文件末尾按相同结构追加即可。

## 八、章节审稿模板（偏离检测 + 质量评估）

> 使用场景：写完一章后，将正文或详细摘要输入，让 Claude 同时检查是否偏离大纲，并评估内容质量是否达到番茄平台日更标准。

### 输入项（用户提供）
- **本章正文或详细摘要**：（必填）
- **本章在大纲中对应的节拍**：（如：大纲第47章应为“中忍考试决赛第一场，主角对战萨克”）
- **主角当前实力/状态参照**：（如：查克拉量≈卡卡西三分一，已掌握螺旋丸但只能两发）
- **当前时间线位置**：（如：中忍考试死亡森林第3天）

### 输出模板（Claude 按此结构输出）

#### 一、大纲偏离度检测
- **大纲预设内容**：（用户填写的“本章对应大纲节拍”）
- **实际写成内容**：（Claude 用一句话概括本章实际剧情）
- **偏离等级**：（三选一：无偏离 / 轻度偏离 / 重度偏离）
- **偏离具体说明**：
  - 若偏离，指出是“事件顺序变化”“角色行为更改”“新增/删除情节”还是“整体方向转向”。
  - 若未偏离，说明本章如何精确执行了大纲节拍。
- **是否需修改大纲**：（是/否，并给出理由）
- **如需修改，对大纲的最小改动建议**：（一句话，如：将大纲第48章中“主角在木叶医院”改为“主角在死亡森林边缘”）

#### 二、内容质量评估（番茄平台标准）
##### 1. 开篇衔接与钩子有效性
- 本章前300字是否平滑承接了上一章的时间线或动作？是否存在段落断层？
- 本章前300字是否包含冲突/悬念/反期待？
- 评分：强 / 中 / 弱
- 若存在断层或钩子较弱，给出一个可替换的开篇过渡句示例。

##### 2. 爽点密度与分布
- 本章出现的正向情绪节点（打脸、突破、获宝、被崇拜、解气）列出清单。
- 统计次数：[ ] 次
- 评价：过高（可能降智）/ 适中 / 不足（建议增加X处）

##### 3. 憋屈/压抑段落长度
- 是否有连续超过500字的压抑、受挫、被误解描写？
- 若有，指出具体段落位置（如“从主角被嘲讽到反击之间间隔约600字”）。
- 建议：压缩中间过渡句，或插入一个即时小爽点对冲。

##### 4. 角色行为一致性
- 主角行为是否符合当前实力与性格设定？
- 原作角色是否出现OOC（角色偏离）？
- 如有问题，指出具体台词或行动并给出修改方向。

##### 5. 章末钩子力度
- 本章结尾最后一句话或最后一段是什么？
- 能否让读者产生“立刻看下一章”的冲动？
- 评分：强力钩 / 普通悬念 / 平淡收尾
- 若平淡，提供一个替代版章末句示例。

##### 6. 文笔与节奏问题速查
- 是否存在连续大段说明性文字（超过200字）？
- 战斗/高潮段落是否节奏拖沓？
- 对话是否过于书面化、缺乏角色辨识度？
- 指出1-3处具体可优化点。

#### 三、综合修改优先级（三选一）
- **可直接发布**：无明显大纲偏离，质量达标。
- **建议微调后发布**：有1-2处小问题，给出5分钟可完成的修改清单。
- **建议大修**：存在重度偏离或连续无爽点，列出修改要点并按重要性排序。
