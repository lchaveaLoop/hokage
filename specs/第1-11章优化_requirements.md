# 第1-11章优化需求文档

> **创建日期**：2026-02-16
> **优化范围**：第1-11章（开篇至波之国任务）
> **优化类型**：中度优化（保留框架，优化表达）

---

## 一、核心原则

**保留**：
- 系统觉醒的开头设定
- 系统提供的核心能力（改命、技能、判定）
- DNF机制的爽点（判定碾压、数值优势）
- 现有的剧情框架和章节结构

**优化**：
- 减少系统提示频率（每章8-10次 → 2-3次）
- 简化系统界面展示
- 去掉游戏化元素（装逼指数、位面舞王等）
- 加强人物刻画和情感描写
- 让系统信息自然融入叙事

**目标**：
> 让读者感受到"这是一个有特殊能力的人在改变命运"，而不是"这是一个游戏角色在刷副本"

---

## 二、具体优化要求

### 2.1 系统表现方式

**原则**：系统是"工具"，不是"主角"

**系统提示使用规范**：

✅ **保留的系统提示**（每章2-3次）：
- 等级提升：【等级提升：Lv.11→Lv.12】
- 技能解锁：【解锁技能：寸拳】
- 改命选择：【检测到改命机会】【代价：等级-1】
- 重要战斗结果：【获得经验：XXX】

❌ **删除的系统提示**：
- 详细的敌人扫描：【姓名：XXX】【能级：XXX】【备注：XXX】
- 战斗判定细节：【判定效果：[绝对倒地]、[防御破碎]】
- 伤害计算：【伤害计算：基础攻击力×120%】
- 装逼指数：【装逼指数：四星半】
- 过于详细的系统分析：【系统分析：真身概率：98.7%】

**改为自然叙事**：
```
原版：【姓名：漩涡鸣人】能级：白（杂兵）。备注：封印存在。

优化：李武的目光扫过鸣人。
     那个金发少年体内封印着某种强大的力量，
     但查克拉混乱得像一团乱麻。
```

### 2.2 战斗描写方式

**原则**：聚焦动作和冲击力，而不是数据

**删除**：
- 详细的判定效果展示
- 伤害计算过程
- 技能触发提示（除非是新技能首次展示）

**保留**：
- 判定优势的核心（用自然方式表达）
- 技能的视觉效果
- 战斗的冲击力

**示例对比**：

❌ **原版**：
```
【技能触发：下段踢】
【判定效果：[绝对倒地]、[防御破碎]、[击退]】
【伤害计算：基础攻击力×120% × 暴击150% = 180%基础伤害】
```

✅ **优化后**：
```
李武的右腿扫出，带着金色的念气光芒。
那一瞬间，他能感觉到——这一击，对方无法防御。
水木的身体失去平衡，重重摔在地上。
```

### 2.3 人物对话优化

**李武的对话风格**：
- 简洁、直接
- 对小李时偶尔温和
- 不过度装逼
- 有真实的情感

❌ **过于装逼**：
```
"所谓的'规则'，只是弱者用来自我安慰的借口。
当你拥有绝对的力量时，规则就只是你脚下的台阶。"
```

✅ **真实自然**：
```
"规则是死的，人是活的。"
李武看着小李，"你不会忍术，那就走体术的路。
找到适合自己的方式，比遵守别人的规则更重要。"
```

**小李的对话风格**：
- 热血、执着
- 对哥哥用敬语
- 有自卑但不放弃
- 真实的情感表达

✅ **保持特色**：
```
"哥哥，我...我真的能毕业吗？"
小李握紧拳头，"我连分身术都用不出来..."

"我会努力的！"小李的眼睛亮了起来，
"就算不会忍术，我也要成为优秀的忍者！"
```

### 2.4 情感描写加强

**每章必须包含**：
- 至少1次兄弟互动场景
- 李武对小李的关心（可以是细节）
- 小李对李武的崇拜或感激

**情感表达方式**：
- 用行动展现，而不是说教
- 用细节展现，而不是直白陈述
- 用对话展现，而不是内心独白

**示例**：
```
李武揉了揉小李的头，这是他少有的温柔动作。
"别担心，有哥哥在。"

小李抬起头，眼眶有些泛红，但很快露出笑容。
"嗯！我相信哥哥！"
```

## 三、章节优化重点

### 第1章：深渊闪光与系统觉醒

**保留**：
- 系统觉醒的设定
- 深渊闪光的开场

**优化**：
- 减少系统界面的详细展示
- 加强李武对小李的关心描写
- 系统觉醒后，聚焦于"这能帮我保护小李"而不是"我变强了"

**系统提示**：2次
1. 系统觉醒提示
2. 初始属性展示（简化）

### 第2章：血条可视化与初体验

**保留**：
- 血条可视化的设定
- 李武测试能力

**优化**：
- 减少系统界面展示
- 加强李武与小李的互动
- 血条可视化用更自然的方式表达

**系统提示**：2次
1. 血条功能解锁
2. 技能解锁提示

### 第3章：毕业考与血条蒸发

**保留**：
- 毕业考试的剧情
- 李武击败水木

**优化**：
- 大幅减少系统提示（从10+次降到2次）
- 去掉"装逼指数"
- 战斗描写聚焦于动作
- 加强李武"为小李铺路"的动机

**系统提示**：2次
1. 战斗结束，获得经验
2. 等级提升（如果有）

### 第4-11章：波之国任务

**保留**：
- 现有剧情框架
- 与第七班的互动
- 再不斩和白的战斗

**优化**：
- 每章系统提示控制在2-3次
- 加强与队友的情感互动
- 白和再不斩的悲剧触动李武
- 为后续"改命"主题做铺垫

## 四、设定调整

### 4.1 系统名称

**原版**：位面舞王系统
**问题**：过于中二，容易劝退读者

**优化方案**（三选一）：
1. 不给系统命名，只称为"系统"
2. 改为"直觉系统"
3. 改为"战斗本能系统"

**建议**：方案1（不命名），保持神秘感

### 4.2 删除的设定

❌ **必须删除**：
- 装逼指数
- 位面舞王称号
- 过于详细的系统分析（如"真身概率98.7%"）

✅ **保留但简化**：
- 能级颜色（白、蓝、紫、金）→ 保留，但不频繁展示
- 血条可视化 → 保留，但用自然方式表达
- 判定优势 → 保留核心，但不展示详细数据

---

## 五、验收标准

### 5.1 系统存在感

**标准**：系统提供能力，但不刷存在感

**检查点**：
- [ ] 每章系统提示不超过3次
- [ ] 没有"装逼指数"等劝退设定
- [ ] 战斗描写聚焦于动作，而不是数据
- [ ] 系统信息自然融入叙事

### 5.2 人物刻画

**标准**：人物有真实的情感和性格

**检查点**：
- [ ] 李武有温柔的一面（对小李）
- [ ] 对话真实，不过度装逼
- [ ] 每章至少1次兄弟互动
- [ ] 小李的性格鲜明

### 5.3 叙事流畅度

**标准**：阅读流畅，不被系统提示打断

**检查点**：
- [ ] 系统提示不打断叙事节奏
- [ ] 战斗描写有冲击力
- [ ] 情感描写自然
- [ ] 对话推进剧情
