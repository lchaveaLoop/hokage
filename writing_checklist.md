# 每章写作检查清单

> 本清单提取自《创作手册》第2.1节,用于日常写作时的快速检查。

---

## 写作前检查 (5分钟)

- [ ] **确认章节位置** - 查看chapter_list.json,确认本章在整体剧情中的位置
- [ ] **检查衔接点** - 查看上一章结尾,确保本章开头自然衔接
- [ ] **确定核心剧情** - 本章的1-2个主要事件是什么?
- [ ] **设计爽点** - 本章至少2-3个爽点(数值碾压/技能震撼/改命成功/情感升华)
- [ ] **确定情感节点** - 本章的1个情感高潮或铺垫是什么?

---

## 写作中检查 (90-120分钟)

### 章节结构

- [ ] **开场** (100-200字) - 承接上章或设置悬念
- [ ] **主体** (1,600-2,000字) - 核心剧情展开,包含2-3个场景
- [ ] **结尾** (200-300字) - 留钩子或情感升华

### 系统提示

- [ ] **频率控制** - 每章3-5次系统提示,不超过8次
- [ ] **格式统一** - 使用【】或「」格式
- [ ] **避免刷屏** - 不要连续多次系统提示

### 场景构建

- [ ] **场景1** - 地点明确,事件清晰,字数200-400字
- [ ] **场景2** - 主要场景,字数800-1,200字
- [ ] **场景3** - 收尾场景,字数400-600字

### 对话比例

- [ ] **对话占比** - 30-40%为佳
- [ ] **叙述占比** - 40-50%
- [ ] **描写占比** - 20-30%

---

## 写作后检查 (10分钟)

### 字数检查

- [ ] **目标字数** - 2,200字(±300字)
- [ ] **灵活范围** - 1,900-2,500字
- [ ] **不足处理** - 如果<2,000字,增加围观者反应/敌人心理/环境描写
- [ ] **超出处理** - 如果>2,500字,删减重复描写/简化战斗过程

### 爽点检查

- [ ] **爽点密度** - 至少2个明确爽点
- [ ] **爽点类型** - 数值碾压/技能震撼/改命成功/情感升华/围观震撼
- [ ] **爽点分布** - 均匀分布,不要集中在开头或结尾

### 质量检查

- [ ] **对话比例** - 30-40%为佳
- [ ] **系统提示** - 格式统一,频率合理
- [ ] **技能展示** - 本章展示的DNF技能是否清晰?
- [ ] **情感节奏** - 本章的情感节点是否到位?
- [ ] **兄弟情感** - 是否体现了"守护弟弟"的核心主题?

### 剧情检查

- [ ] **剧情推进** - 本章是否推进了主线剧情?
- [ ] **角色成长** - 本章是否展现了角色成长?
- [ ] **伏笔铺垫** - 本章是否为后续剧情埋下伏笔?
- [ ] **结尾钩子** - 本章结尾是否吸引读者继续阅读?

---

## 战斗章节专用检查

### 战斗场景结构 (3-5-8原则)

#### 3段式结构
- [ ] **开场试探** (300字) - 展示敌我差距,敌人轻视→主角冷静分析
- [ ] **主体碾压** (800字) - 技能连招+数值展示+判定说明+敌人心理变化
- [ ] **结束震撼** (200字) - 围观者反应+敌人绝望+主角淡定离场

#### 5个必备要素
- [ ] **敌人能级显示** - 系统扫描:【蓝色·精英级】【紫色·BOSS级】
- [ ] **数值对比展示** - 属性差距:力量2000 vs 力量500
- [ ] **技能判定说明** - 机制解释:【抓取判定】【优先级:超越霸体】
- [ ] **伤害计算过程** - 数值爽点:暴击!寸拳×2.5倍!9999伤害!
- [ ] **敌人心理变化** - 轻视→震惊→恐惧→绝望

#### 8个避免事项
- [ ] 避免纯力量对轰,必须体现DNF机制特色
- [ ] 避免战斗过长(单场战斗不超过1,500字)
- [ ] 避免重复描写(同一技能变换角度描写)
- [ ] 避免忽略围观者(旁观者视角增强代入感)
- [ ] 避免敌人无脑送(给敌人合理的战术和反应)
- [ ] 避免系统提示刷屏(每场战斗3-5次系统提示)
- [ ] 避免忽略环境(战斗对环境的破坏增强视觉冲击)
- [ ] 避免结束仓促(给足围观者震撼反应的篇幅)

---

## 日常章节专用检查

### 日常场景内容

- [ ] **兄弟互动** - 李武与小李的日常对话/训练/生活
- [ ] **队友互动** - 与第四班/第七班的交流
- [ ] **训练场景** - 技能练习/属性提升/系统提示
- [ ] **信息收集** - 情报获取/任务接取/传闻听取

### 日常章节作用

- [ ] **缓冲情感** - 避免高潮疲劳
- [ ] **铺垫剧情** - 为下一个高潮做准备
- [ ] **丰富角色** - 展现日常生活和关系

---

## 情感章节专用检查

### 情感类型

- [ ] **兄弟情深** - 李武与小李的情感时刻
- [ ] **改命时刻** - 改变原著悲剧的情感爆发
- [ ] **守护决心** - 展现主角的守护意志
- [ ] **价值观体现** - 主角的价值观和选择

### 情感节奏

- [ ] **铺垫充分** - 情感爆发前有足够铺垫
- [ ] **高潮到位** - 情感高潮点足够震撼
- [ ] **余韵留存** - 情感高潮后有余韵

---

## 节奏检查 (每5章检查一次)

### 黄金节奏比例: 3战斗 + 1日常 + 1情感

- [ ] 最近5章是否都是战斗? (是→加入日常章)
- [ ] 最近5章是否都是日常? (是→加入战斗章)
- [ ] 最近25章是否有情感高潮? (否→安排改命/觉醒)
- [ ] 最近10章是否有兄弟互动? (否→安排兄弟场景)

---

## 提交前最终检查

- [ ] **文件命名** - 第XX章.md (使用中文数字)
- [ ] **文件位置** - 保存到正确的分卷目录
- [ ] **Git提交** - git add . && git commit -m "完成第XX章: [标题]"
- [ ] **更新进度** - 更新chapter_list.json中的状态为"completed"
- [ ] **更新日志** - 在writing_progress.txt中记录本次会话

## 快速参考

### 爽点类型速查

**数值碾压类** (最常用):
- 等级差距展示 (Lv.50 vs Lv.20)
- 属性对比 (力量2000 vs 力量500)
- 伤害计算过程 (暴击!9999伤害!)
- 秒杀效果 (一击必杀)

**技能震撼类**:
- 新技能首次展示
- 觉醒技能登场
- 技能连招组合
- 判定优先级碾压

**改命成功类** (情感爆发):
- 原著角色存活
- 悲剧被改写
- 代价承受展示
- 众人震惊反应

**情感升华类**:
- 兄弟情深时刻
- 守护决心展现
- 价值观体现
- 角色成长瞬间

**围观震撼类**:
- 旁观者视角
- 众人议论
- 名声传播
- 实力认可

## 常见问题处理

### 字数不足 (<2,000字)
- 增加围观者反应 (200-300字)
- 增加敌人心理活动 (100-200字)
- 增加环境描写 (100-200字)
- 增加系统提示详细信息 (100字)
- 增加主角内心独白 (200-300字)

### 字数过多 (>2,500字)
- 删减重复描写
- 简化战斗过程 (多个技能合并描写)
- 删减次要角色对话
- 简化环境描写
- 将部分内容移到下一章

### 爽点不足
- 增加数值对比展示
- 增加围观者震撼反应
- 增加敌人心理变化
- 增加技能视觉效果描写

### 节奏单调
- 检查最近5章的类型分布
- 按照3战斗+1日常+1情感调整
- 增加小冲突或小惊喜

---

**使用建议**:
- 每次写作前打印本清单
- 写作过程中逐项检查
- 写作后再次完整检查
- 每5章进行一次节奏检查
