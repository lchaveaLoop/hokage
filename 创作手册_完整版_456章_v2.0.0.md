1. 先读本文件，确认主题、冲突、系统硬规则、时间锚点。

## 第一章：故事内核

### 1.1 一句话卖点

穿越火影的 DNF 玩家李武，用念气与数值体系打破血脉垄断，在一次次“能不能改命”的抉择里，证明没有血继限界的人也能走到世界顶点。

### 1.2 读者承诺

- 这是一本先靠“机制破局”吸引人，再靠“改命代价”留下人的书。
- 爽点不是单纯秒人，而是“看穿、反制、碾过宿命逻辑”。并且必须包括极致的降维打击——用游戏底层的绝对判定（如：无敌帧、霸体护甲、强制抓取），直接碾碎原著忍术体系的逻辑闭环。
- 悲情不是为了虐，而是为了让每一次出手都更值钱。
- 归家不是逃离，而是完成一场证明之后的告别。

### 1.3 三层核心冲突

1. 个体层：DNF 机制 vs 火影查克拉体系  
   李武的强，不来自血脉传承，而来自机制理解、数值成长、战斗判断。
2. 社会层：平民努力 vs 血脉垄断  
   写轮眼、白眼、尸骨脉、大筒木血脉，本质都是阶层门槛。
3. 终局层：人的创造性 vs 既定神格秩序  
   李武最终要打破的，不只是强者本身，而是“只有那种人才能强”的叙事前提。

### 1.4 情绪基调

- 战斗场景：快、狠、准，重在机制碾压。主角永远从容不迫，面对任何敌人都带有高维降维打击的散漫感与绝对压制。
- 日常场景：黑色幽默、高逼格吐槽。不再是苦情穷兄弟，而是满级大佬降维带飞，毒舌腹黑与极其护短的特质并存。
- 处事原则：内圣外王，斩草除根。面对高层的威胁或恶霸的试探，绝不隐忍不圣母，不出手则已，出手必摧枯拉朽。
- 全书底色：表面玩世不恭腹黑吐槽，内心稳守底线，用绝对的暴力数值摧毁虚伪深沉的忍界宿命论。

### 1.5 结局基调

- 结局保留为 `800章` 体量，不再同时保留 `780完结 / 791完结 / 800再结语` 多套口径。
- 终局必须保住四层情绪：
  - 证明：没有血脉也能登顶。
  - 代价：不是所有人都救得回来。
  - 传承：李武离开后，这条路不会断。
  - 告别：归家是完成，不是抽身。

### 1.6 前150章角色优先级

前 `150章` 只重点吃透四条 A 级线，避免中前期抢戏过多：

| 线别 | 定位 | 前150章任务 |
|------|------|-------------|
| 小李线 | 情感锚点 | 让“非血脉也能变强”落到人身上 |
| 宁次线 | 主题镜子 | 让宿命论与自由意志持续对撞 |
| 白线 | 第一块悲情试金石 | 让“改命不是想救就能救”第一次见血 |
| 佐助/团藏线 | 主线压力源 | 一条给主角价值观对照，一条给体制压迫实体化 |

鼬、长门、带土、绝在前150章只做远期伏笔，不喧宾夺主。

## 第二章：本书运营作战表

### 2.1 开书任务

- 前 `3章` 要完成：主角立住、系统可见、兄弟线成立。
- 前 `10章` 要完成：世界残酷、主角有路、小李值得追。
- 前 `30章` 要完成：木叶高层开始注意李武，读者确认这不是普通升级流。
- 前 `50章` 要完成：波之国大高潮，用白的牺牲把“改命代价”打进读者心里。

### 2.2 标题口径

标题优先级固定为：`结果 > 冲突 > 情绪 > 概念`。

| 场景 | 不要写法 | 建议写法 |
|------|----------|----------|
| 能力展示 | 念气的运用 | 念气升级，一拳捏碎石头 |
| 战斗高潮 | 白的魔镜冰晶 | 魔镜冰晶里，我只找到了白 |
| 情感章 | 津奈美的故事 | 她问：忍者拿钱的时候，平民在哪 |
| 命运抉择 | 命运节点 | 我第一次看见“白会死” |

### 2.3 每10章爆点配比

每 `10章` 至少稳定交付以下四类爆点里的三类：

| 类型 | 占比 | 说明 |
|------|------|------|
| 机制爽点 | 3 | 看穿、反制、数值压制、打脸 |
| 情感爽点 | 2 | 兄弟线、师徒线、被认可、被追随 |
| 世界爽点 | 1 | 高层关注、制度裂缝、血脉门槛被揭开 |
| 命运爽点 | 1 | 看到能不能救、为什么救不了、以后怎么救 |

### 2.4 卷尾断章模板

卷尾断章只准用以下四种：

1. 看见更大的敌人  
   例：李武意识到自己刚打赢的是局部，真正盯上他的人还没出手。
2. 看见更重的代价  
   例：白死了，但系统告诉他“能救”，只是现在救不起。
3. 看见关系转折  
   例：宁次第一次动摇，佐助第一次把李武当成真正对手。
4. 看见下一层力量门槛  
   例：逻辑锁可解析，但当前等级不够。

### 2.5 评论引导口径

每逢高潮章或卷尾章，评论区引导优先写成“站队题”和“代价题”，不要写空泛求互动：

- “这一章如果是你，你救不救白？”
- “你觉得宁次最先动摇的是哪一句？”
- “如果没有血继，李武这条路到底算不算正路？”
- “你们更想先看佐助线，还是先看团藏出手？”

### 2.6 写作禁忌

- 不要把系统写成万能旁白。
- 不要为了番茄节奏把人物说话写油。
- 不要让每一章都靠“震惊体”起标题。
- 不要在中前期让过多原著名场面排队抢戏。

## 第四章：系统 / 数值 / 改命硬规则

### 4.1 总原则

- 系统负责“给主角一条能走通的路”，不负责替主角做选择。
- 面板只给结论、边界和代价，不替代战斗过程。但在高潮战斗与改命节点时，必须极其高调地弹出系统 UI 提示（如 `【破招判定成功】`、`【霸体护甲生成】`），给读者最直接的数值与机制碾压爽感。
- 能力成长必须先有现实反馈，再给系统提示。
- 任何高价值改命，都必须伴随真实代价，且代价不能被当章抹平。

### 4.2 等级档位规则

| 等级段 | 能做到什么 | 做不到什么 |
|--------|------------|------------|
| Lv.1-10 | 感知、基础破局、止痛、近身小胜 | 不能硬改命，不能拆高阶逻辑锁 |
| Lv.11-20 | 单兵 D/C 级任务稳定，能看穿低阶忍术破绽 | 不能正面扛上忍级压制 |
| Lv.21-35 | 中忍层面对抗、短时爆发、机制反制成熟 | 不能长期连续越级碾压 |
| Lv.36-50 | 上忍战边缘、能干预部分术式运行，一觉前夜 | 不能稳定改写大型因果链 |
| Lv.51-65 | 武神线成型，可解析并干预笼中鸟级逻辑锁 | 不能无代价连救多人 |
| Lv.66-85 | 晓级战主力，改命开始具备可操作性 | 不能无视组织战与情报战 |
| Lv.86-100 | 极武圣阶段，可对尾兽/六道以下战场形成决定性影响 | 不能直接抹掉世界规则 |
| Lv.101-120 | 归元到归真，进入终局规则战 | 仍然不能把所有遗憾归零 |

### 4.3 逻辑锁规则

- 笼中鸟在本书里不是“玄学咒印”，而是“可解析、可干预、可破解”的逻辑锁。
- 固定门槛：
  - `Lv.10`：看见结构。
  - `Lv.50`：短时干预运行。
  - `Lv.65`：彻底破解单体笼中鸟。
- 逻辑锁破解不能通用于一切封印。每种锁都必须重新分析。
- 一次成功干预，不等于体系层面已经胜利。真正的胜利是制度被改。

### 4.4 命运节点规则

- 命运节点只在“原著强因果链 + 主角已触碰核心人物”两个条件同时成立时弹出。
- 不是每次濒死都弹节点，只有会改变卷级走向的死亡/转折才弹。当这类极端悲情节点出现时（如再不斩牺牲、白的命运转折等），系统必须用极强烈的视觉提示介入（如：`【警告！S级命运节点挂起】`、`【启动最后生机锁定干预】`），以此来稳住读者心理底线，防止弃书。
- 节点只给三种选项：
  - 立即干预
  - 延后干预
  - 不干预
- 节点越大，代价越高；链路越长，掉级越狠。

### 4.5 改命代价规则

- 改命默认代价是“临时掉级 + 念气紊乱 + 后续战斗风险提升”。
- 掉级不是永久废号，但恢复必须靠后续战斗、训练、或高压突破，不允许睡一觉满血回档。
- 连续改命会叠加副作用：
  - 技能锁死
  - 感知失真
  - 念气回路迟滞
- 改命成功，不等于情感问题解决。人活下来了，账还在。

### 4.6 作者禁区

- 禁止“作者想救谁，系统就临时开后门”。
- 禁止“主角等级不够，但情绪到了，所以强行翻盘”。
- 禁止“一次解析，后续所有类似问题都秒懂”。
- 禁止“重要角色都被救下，遗憾只靠配角承担”。

## 第七章：三层文档结构

### 7.1 文档分工

| 文件 | 定位 | 用法 |
|------|------|------|
| `创作手册_完整版_456章_v2.0.0.md` | 世界圣经 + 总索引 | 定调、定边界、查规则 |
| `创作手册_1-150章生产大纲.md` | 生产文档 | 直接指导落稿 |
| `创作手册_151-800章远期里程碑.md` | 远期路线图 | 控长线，不写伪细纲 |

### 7.2 执行原则

- 一个章节号，只允许承担一种主功能。
- 一个阶段，只允许有一条清晰主线推进。
- 中前期写作优先级永远高于远期设定堆积。
- 当正文和远期大纲冲突时，先校正远期大纲，不回头破坏已落地的优质正文。

### 7.3 当前写作建议

- 眼下实际创作以波之国篇与 `第51-55章` 战后收束为最高优先级。
- 需要继续精修单章时，以 `docs/plans/2026-03-31-前30章精修写作标准.md` 的信息密度标准执行。
- 新增设定前先问三件事：是不是服务当前卷、是不是会制造新冲突、是不是会让后文更好写。
